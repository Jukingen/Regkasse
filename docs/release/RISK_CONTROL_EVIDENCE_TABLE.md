# Risk → Kontrol → Kanıt → Kalan Boşluk

Bu tablo, teknik review sonrası yapılan kontrolleri ve kanıtları tek yerde toplar. Kolonlar: **Risk**, **Mevcut kontrol / yeni kontrol**, **Kod dosyası**, **Test / SQL / doküman kanıtı**, **Kanıt seviyesi**, **Kalan açık risk**.

## Kısaltmalar / notlar

- **Kanıt seviyesi:** Yüksek = gerçek PostgreSQL/Testcontainers veya production-benzeri SQL ile doğrulama; Orta = test veya script var ama kapsam/ortam sınırlı; Düşük = gözlemsel metrik veya doküman.
- **Yeni** = bu oturumda eklenen veya netleştirilen kontrol/doküman (backlog, validation SQL offline index, legacy payload-hash doc, test summary).
- Tüm kod yolları `backend/` altında; dokümanlar `docs/release/` altında.
